# 데이터 적재 파이프라인 (사전작업)

`files0/10-backend-spec.md` §4의 적재 파이프라인을 실 데이터로 실행해 `schema.sql`에
바로 넣을 수 있는 시드 SQL을 만든다. Spring Boot 앱과 무관하게 독립 실행된다.

## 실행

원본 CSV(`식품_일반음식점_경기성남시_2016~.csv`, EUC-KR)가 이 폴더에 그대로 들어있다.

```
npm install
npm run parse
```

원본 파일 기준: 성남시 전역·2016~2026, 30,453행 중 실데이터 9,522행(나머지는 파일
끝 공백 패딩행).

출력: `out/data.sql`(INSERT문), `out/summary.json`(적재 통계). `out/`에는 위 CSV로
한 번 실행한 결과물이 예시로 이미 들어있다(재실행하면 덮어씀).

### 다른 데이터셋에 재사용하기

이 스크립트는 "성남시 수정구 일반음식점" 전용이 아니라 인허가 데이터 전반에 통하는
파이프라인 뼈대(로드→필터→정제→PNU 생성→자리/물건 분리→통계→SQL)다. 다른
지자체·업종 데이터로 바꾸려면 `src/parse.mjs`에서 이 세 부분만 손보면 된다:

- `DONG_CODES` — 대상 지역의 법정동코드 표(코드 자체는 항상 code.go.kr 공개자료).
- `JIBUN_RE` — 지번주소 파싱 정규식의 구 이름(`수정구`) 부분.
- `SRC_CSV`, CSV 컬럼명 상수(`인허가일자`, `사업장명` 등) — 원본 컬럼 체계가 다르면 여기만.

나머지(정제 P-1~P-4, PNU 조립, D-U1 물건 분리, 통계 계산, SQL 이스케이프)는
그대로 재사용 가능.

## 처리 결과 (2026-07-08 기준 실행)

- 수정구 필터: 2,709행
- 제외: 0건 (영업상태·날짜순서·주소파싱 전부 통과)
- 좌표 없음: 29개 자리(지도 마커 제외 대상, VWorld 지오코딩 미실행 — 아래 참고)
- 자리(site) 1,192개 · 물건(unit) 2,591개 · 이력(tenancy) 2,709건

## schema.sql 수정

원본 `schema.sql`에는 `unit` 테이블이 없고 `tenancy_record.site_pnu`로 바로 연결돼
있었다. 그런데 `10-backend-spec.md` §3.1(D-U1)은 이미 "물건 단위 확정"으로 명시돼
있고 API 계약도 `unitId`를 키로 쓴다 — `unitId`가 저장될 테이블이 없으면 계약이
성립하지 않는 모순이었다. `unit`, `unit_statistics` 테이블을 추가하고
`tenancy_record`를 `unit_id` 참조로 바꿨다(상세는 schema.sql 주석 참고).

## 물건 분리 로직에서 발견·수정한 문제

스펙 3.1 규칙 5(상세주소 있으면 그 문자열을 물건 식별자로 우선 사용)를 그대로
구현했더니, 상세주소 문자열이 같아도 **영업 기간이 겹치는 서로 다른 상호**가 한
물건으로 합쳐지는 사례가 나왔다(예: 배달전용 공유주방 — 같은 호수 문자열로 신고된
가게 5곳이 전부 같은 날짜에 개업해 동시 영업 중). "물리적으로 한 칸엔 동시에 두
가게가 영업할 수 없다"는 규칙 2의 전제를 어기는 결과라, 상세주소 유무와 무관하게
겹침 분리(규칙 2~4)를 항상 적용하도록 일반화했다. 이 경우 같은 상세주소를 쓰는
물건들은 `라벨 (1)`, `라벨 (2)`처럼 번호를 붙여 구분한다.

## 의도적으로 미룬 것 (ponytail)

- **VWorld 지오코딩 미연동**: API 키 발급이 필요해 이번 사전작업 범위 밖. 원본
  CSV의 좌표(X,Y, EPSG:5174 추정 — 표본 좌표 역변환으로 확인)만 WGS84로 변환해
  채웠다. 좌표 없는 29개 자리는 `latitude/longitude = NULL`로 남겨뒀다(지도 마커
  제외, 목록엔 노출). 실 서비스 전환 시 `toWgs84` 폴백으로 VWorld 호출 추가.
- **분당구·중원구 미처리**: 원본 CSV는 성남시 전역(분당구 5,007행·중원구 1,803행
  포함)이지만 스펙이 수정구로 스코프를 명시해 그대로 따름. 확장 시 `DONG_CODES`
  테이블만 늘리면 됨(법정동코드는 이미 전국 공개자료라 동일한 방식으로 조회 가능).
- **소재지면적 등 부가 필드 미반영**: 원본에 면적·보증액·월세액 컬럼이 있지만
  현재 API 계약(`00-api-contract-v1.md`)에 해당 필드가 없어 시드에서 제외.
  프론트 상세 화면의 "계약·주변상권 정보"는 여전히 예시값이며, 계약에 필드를
  추가하면 이 파이프라인에 한 줄씩만 추가하면 된다.

## 재실행 방법

CSV가 갱신되면 `npm run parse`만 다시 돌리면 `out/data.sql`이 통째로 재생성된다
(멱등— 같은 입력이면 같은 출력).
